# 校园快递管理系统 - 前端开发计划（本科毕业设计）

## 一、项目概述

### 1.1 项目信息
- **项目名称**: 校园快递管理系统
- **项目类型**: 本科毕业设计
- **技术栈**: Vue3 + Vite + TypeScript + Element Plus + Pinia + Vue Router
- **后端接口**: Spring Boot RESTful API
- **开发目标**: 功能完整、代码清晰、便于答辩

### 1.2 开发原则
- ✅ 所有组件仅服务对应业务模块，功能简单直观
- ✅ 每个页面顶部添加功能说明和对应毕业设计模块
- ✅ 每个函数添加必要注释，复杂逻辑说明用途
- ✅ 错误提示保持简单（如 401 / 404 / 500），直接在组件中处理
- ✅ 目录以业务模块为中心，不做通用组件库
- ✅ 界面清晰简单，使用 Element Plus 默认样式

### 1.3 功能限制（禁止做）
- ❌ Axios 封装、统一错误处理、请求拦截器
- ❌ 懒加载路由、路由懒加载
- ❌ 通用表格/表单组件（PageTable、PageForm、PageModal）
- ❌ 按钮级权限、动态路由、权限指令
- ❌ 国际化、多主题切换、复杂性能优化
- ❌ 企业级特性（Tailwind、多标签页、微前端等）

---

## 二、技术栈说明

### 2.1 核心技术
- **Vue3**: 渐进式 JavaScript 框架
- **Vite**: 新一代前端构建工具
- **TypeScript**: JavaScript 的超集，提供类型安全
- **Element Plus**: Vue 3 组件库
- **Pinia**: Vue 状态管理库
- **Vue Router**: Vue 官方路由管理器
- **ECharts**: 图表库（用于数据统计展示）

### 2.2 技术选择理由（答辩用）
- **Vue3**: 当前主流前端框架，生态成熟，适合快速开发
- **Vite**: 开发体验好，构建速度快
- **TypeScript**: 提供类型检查，减少运行时错误
- **Element Plus**: 组件丰富，文档完善，适合快速搭建后台系统
- **Pinia**: Vue3 官方推荐的状态管理库，API 简洁
- **Vue Router**: 官方路由管理器，与 Vue3 完美集成
- **ECharts**: 功能强大的图表库，适合数据可视化

### 3.2 目录设计说明（答辩用）
- **api/**: 按业务模块划分，每个模块对应一个后端 Controller，直接使用 fetch 调用
- **components/**: 按业务模块划分，每个组件仅服务于对应业务，不做通用组件
- **views/**: 按业务模块划分，每个页面对应一个功能模块
- **layouts/**: 简单的主布局，包含侧边栏和顶部导航
- **router/**: 静态路由配置，不使用动态路由生成，不使用懒加载
- **store/**: 简单的用户状态管理，仅存储登录信息和用户信息
- **types/**: TypeScript 类型定义，提供类型安全
- **utils/**: 工具函数，仅包含必要的验证函数

## 四、功能模块设计

### 4.1 登录页面（views/login/index.vue）

#### 功能说明
- 毕业设计模块：用户认证模块
- 用户名/密码登录
- 登录校验（用户名和密码不能为空）
- 管理员/普通用户角色区分
- 登录成功后跳转到首页

#### 实现要点
- 使用 Element Plus 的 Form 组件
- 表单验证规则：用户名必填、密码必填
- 登录成功后存储 token 和用户信息到 localStorage
- 根据用户角色跳转到不同页面（管理员跳转到管理后台，普通用户跳转到用户首页）
- 直接使用 fetch 调用后端登录接口
- 错误处理：直接在组件中处理，不使用统一拦截器

#### 答辩要点
- 表单验证使用 Element Plus 的 Form Rules
- 登录状态使用 Pinia 存储
- Token 存储在 localStorage 中，实现持久化
- 直接使用 fetch 调用接口，不使用 Axios 封装

### 阶段二：基础框架搭建（1天）

#### 2.1 配置路由
- 创建路由配置文件（router/index.ts）
- 配置静态路由（登录、首页、各管理页面）
- 配置路由守卫（登录验证 + 角色区分）
  - 未登录用户：只能访问登录页，其他页面跳转到登录页
  - 管理员用户：可访问所有管理页面（快递管理、取件记录、快递站管理、用户管理）
  - 普通用户：可访问用户首页和我的快递页面
- 不使用动态路由生成
- 不使用懒加载

#### 答辩要点（路由守卫）
- 使用 Vue Router 的 beforeEach 钩子实现路由守卫
- 通过判断 localStorage 中的 token 和用户角色实现权限控制
- 管理员和普通用户有不同的路由访问权限，展示权限控制思路
- 未登录用户自动跳转到登录页，已登录用户访问登录页自动跳转到首页

#### 2.2 配置状态管理
- 创建 Pinia Store（store/index.ts）
- 创建用户状态模块（store/modules/user.ts）
- 实现登录、登出、获取用户信息等方法

#### 2.3 创建主布局
- 创建主布局组件（layouts/MainLayout.vue）
- 实现侧边栏（菜单导航）
- 实现顶部导航（用户信息、退出）

#### 2.4 配置全局样式
- 创建全局样式文件（assets/styles/index.css）
- 定义基础样式变量（主题颜色、间距、字体等）
  - 主题颜色：主色、成功色、警告色、危险色、信息色
  - 间距：小、中、大间距
  - 字体：字体大小、行高、字重
- 保证整体风格统一，同时不影响毕设简洁原则
- 不使用复杂的 CSS 预处理器（如 Sass/Less）

#### 答辩要点（全局样式）
- 使用 CSS 变量定义主题颜色，便于统一管理
- 基础样式变量保证整体风格统一
- 样式简洁，不使用复杂的 CSS 预处理器
- 符合毕设简洁原则，同时保证界面美观

## 六、接口对接说明

### 6.1 接口列表

#### 认证接口
- POST /api/auth/login - 用户登录
- GET /api/auth/info - 获取用户信息
- POST /api/auth/logout - 用户登出

#### 快递管理接口
- GET /api/express/list - 获取快递列表
- POST /api/express/add - 添加快递
- PUT /api/express/update - 更新快递
- DELETE /api/express/delete/{id} - 删除快递

#### 取件记录接口
- GET /api/pickup/list - 获取取件记录列表
- POST /api/pickup/add - 添加取件记录
- PUT /api/pickup/update - 更新取件记录
- DELETE /api/pickup/delete/{id} - 删除取件记录

#### 快递站管理接口
- GET /api/station/list - 获取快递站列表
- POST /api/station/add - 添加快递站
- PUT /api/station/update - 更新快递站
- DELETE /api/station/delete/{id} - 删除快递站

#### 用户管理接口
- GET /api/user/list - 获取用户列表
- POST /api/user/add - 添加用户
- PUT /api/user/update - 更新用户
- DELETE /api/user/delete/{id} - 删除用户

### 6.2 接口调用方式
- 直接使用 fetch 调用接口
- 不使用 Axios 封装
- 不使用统一错误处理
- 错误处理直接在组件中处理

### 6.3 错误处理
- 401: 未登录，跳转到登录页
- 403: 无权限，提示无权限
- 404: 接口不存在，提示接口错误
- 500: 服务器错误，提示服务器错误
- 其他: 提示具体错误信息

#### 6.4 接口错误模拟（开发阶段）
- 在开发阶段适当模拟 401/403/404/500 错误场景
- 模拟未登录状态（清除 localStorage 中的 token）
- 模拟无权限访问（使用普通用户访问管理员接口）
- 模拟接口不存在（调用错误的接口路径）
- 模拟服务器错误（后端抛出异常）
- 答辩演示时可以展示错误处理机制，更直观

#### 答辩要点（错误处理）
- 直接在组件中处理错误，不使用统一拦截器
- 使用 Element Plus 的 Message 组件提示错误信息
- 401 错误自动跳转到登录页
- 其他错误显示具体错误信息
- 开发阶段模拟各种错误场景，确保错误处理机制完善
