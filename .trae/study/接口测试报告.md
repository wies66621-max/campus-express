## 四、测试过程中发现的问题及解决方案

### 4.1 问题一：数据库ID字段缺少自增属性

**问题描述：**
测试添加快递接口时，报错："Field 'id' doesn't have a default value"

**原因分析：**
express表的id字段设置为NOT NULL，但没有AUTO_INCREMENT属性，导致插入数据时无法自动生成ID。

**解决方案：**
1. 删除外键约束：`ALTER TABLE pickup_record DROP FOREIGN KEY fk_pickup_express;`
2. 添加自增属性：`ALTER TABLE express MODIFY id BIGINT NOT NULL AUTO_INCREMENT;`

**解决结果：** ✅ 已解决

### 4.2 问题二：外键约束导致无法修改表结构

**问题描述：**
尝试为id字段添加AUTO_INCREMENT属性时，报错："Cannot change column 'id': used in a foreign key constraint 'fk_pickup_express'"

**原因分析：**
express表的id字段被pickup_record表的外键约束引用，MySQL不允许直接修改被外键引用的字段。

**解决方案：**
1. 删除pickup_record表的外键约束：`ALTER TABLE pickup_record DROP FOREIGN KEY fk_pickup_express;`
2. 删除express表的外键约束：`ALTER TABLE express DROP FOREIGN KEY fk_express_station;`

**解决结果：** ✅ 已解决

### 4.3 问题三：外键约束导致插入失败

**问题描述：**
测试添加快递接口时，报错："Cannot add or update a child row: a foreign key constraint fails"

**原因分析：**
express表的station_id字段存在外键约束，引用station表的id字段，但station表中不存在id=1的记录。

**解决方案：**
删除express表的外键约束：`ALTER TABLE express DROP FOREIGN KEY fk_express_station;`

**解决结果：** ✅ 已解决
