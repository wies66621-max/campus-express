## 🚀 启动后端项目

### 第一步：检查数据库

**1. 确认MySQL服务已启动**

打开命令行，输入：
```bash
mysql -u root -p
```

如果能进入MySQL命令行，说明MySQL服务已启动。

如果提示"Can't connect to MySQL server"，说明MySQL服务未启动。

**启动MySQL服务：**
- Windows：打开"服务"，找到"MySQL"，右键"启动"
- 或者使用命令：`net start mysql`

---

**2. 确认数据库已创建**

在MySQL命令行中，执行：
```sql
SHOW DATABASES;
```

查看是否有`campus_express`数据库。

如果没有，创建数据库：
```sql
CREATE DATABASE campus_express CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

---

**3. 确认表已创建**

在MySQL命令行中，执行：
```sql
USE campus_express;
SHOW TABLES;
```

查看是否有以下表：
- express
- user
- station
- pickup_record
- notice

如果没有表，需要执行SQL脚本创建表。

---

### 第二步：检查配置文件

**1. 打开配置文件**

找到文件：`src/main/resources/application.yml`

**2. 检查数据库配置**

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_express?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的MySQL密码
```

**确认以下信息：**
- 数据库名称是否正确（campus_express）
- 用户名是否正确（通常是root）
- 密码是否正确（你设置的MySQL密码）

---

### 第三步：启动后端项目

**1. 打开命令行**

在项目根目录下打开命令行。

**2. 运行项目**

输入以下命令：
```bash
mvn spring-boot:run
```

**3. 等待启动**

你会看到很多日志输出，等待几秒钟。

**4. 确认启动成功**

如果看到类似以下输出，说明启动成功：
```
Started CampusExpressApplication in 5.234 seconds
```

**5. 访问接口文档**

打开浏览器，访问：
```
http://localhost:8080/doc.html
```

如果能看到接口文档页面，说明后端启动成功。

---

## 🎨 启动前端项目

### 第一步：检查依赖

**1. 打开命令行**

在`campus-express-frontend`目录下打开命令行。

**2. 检查node_modules是否存在**

查看目录下是否有`node_modules`文件夹。

如果没有，需要安装依赖：
```bash
npm install
```

这个过程可能需要几分钟，因为要下载很多npm包。

---

### 第二步：检查配置

**1. 打开配置文件**

找到文件：`campus-express-frontend/src/utils/request.ts`

**2. 检查后端地址**

确认后端地址配置是否正确：
```typescript
const baseURL = 'http://localhost:8080'
```

---

### 第三步：启动前端项目

**1. 打开命令行**

在`campus-express-frontend`目录下打开命令行。

**2. 运行项目**

输入以下命令：
```bash
npm run dev
```

**3. 等待启动**

你会看到类似以下输出：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

**4. 访问前端页面**

打开浏览器，访问：
```
http://localhost:5173/
```

如果能看到登录页面，说明前端启动成功。

## 🔍 常见问题排查

### 问题1：后端启动失败

**症状：**
- 运行`mvn spring-boot:run`后报错
- 提示"Connection refused"或"Access denied"

**可能原因：**
1. MySQL服务未启动
2. 数据库配置错误
3. 数据库不存在
4. 端口被占用

**解决方法：**

**检查MySQL服务：**
```bash
net start mysql
```

**检查数据库配置：**
打开`application.yml`，确认数据库连接信息正确。

**检查数据库是否存在：**
```sql
SHOW DATABASES;
```

**检查端口是否被占用：**
```bash
netstat -ano | findstr 8080
```

如果端口被占用，修改`application.yml`中的端口：
```yaml
server:
  port: 8081
```

---

### 问题2：前端启动失败

**症状：**
- 运行`npm run dev`后报错
- 提示"command not found"或"ENOENT"

**可能原因：**
1. Node.js未安装
2. 依赖未安装
3. 端口被占用

**解决方法：**

**检查Node.js是否安装：**
```bash
node -version
npm -version
```

**安装依赖：**
```bash
npm install
```

**检查端口是否被占用：**
```bash
netstat -ano | findstr 5173
```

如果端口被占用，修改`vite.config.ts`中的端口：
```typescript
server: {
  port: 5174
}
```

---

### 问题3：登录失败

**症状：**
- 输入正确的账号密码后，提示"登录失败"
- 或提示"用户名或密码错误"

**可能原因：**
1. 数据库中没有admin用户
2. 密码加密方式不匹配
3. 后端接口调用失败

**解决方法：**

**检查数据库中是否有admin用户：**
```sql
USE campus_express;
SELECT * FROM user WHERE username = 'admin';
```

如果没有，插入admin用户：
```sql
INSERT INTO user (username, password, real_name, phone, role, create_time, update_time)
VALUES ('admin', '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iAt6Z5EH', '管理员', '13800138000', 'admin', NOW(), NOW());
```

**检查后端接口：**
打开浏览器，访问`http://localhost:8080/doc.html`，测试登录接口。

---

### 问题4：跨域问题

**症状：**
- 浏览器控制台提示"CORS policy"
- 或提示"Access-Control-Allow-Origin"

**可能原因：**
- 前端和后端不在同一个域名
- 后端没有配置跨域

**解决方法：**

**检查后端跨域配置：**
找到`CorsConfig.java`，确认配置正确。

**或在前端配置代理：**
修改`vite.config.ts`：
```typescript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

---

### 问题5：数据库连接失败

**症状：**
- 后端启动时提示"Communications link failure"
- 或提示"Could not create connection to database server"

**可能原因：**
1. MySQL服务未启动
2. 数据库地址错误
3. 数据库端口错误
4. 防火墙阻止

**解决方法：**

**检查MySQL服务：**
```bash
net start mysql
```

**检查数据库地址：**
确认`application.yml`中的地址是`localhost:3306`

**检查防火墙：**
关闭防火墙或添加MySQL端口（3306）到允许列表。

## 💡 学习建议

1. **不要害怕报错**
   - 报错是正常的
   - 仔细看错误信息
   - 使用搜索引擎查找解决方案

2. **一步步排查**
   - 不要急躁
   - 从最简单的地方开始检查
   - 比如先检查服务是否启动

3. **记录问题**
   - 记录遇到的问题
   - 记录解决方法
   - 下次遇到类似问题可以快速解决

4. **多测试几次**
   - 不要测试一次就结束
   - 多测试几次确保稳定
   - 测试各种场景
