## 🛠️ 技术栈详解

### 后端技术栈

#### 1. **Spring Boot 3.x**

**这是什么？**
- Java开发框架
- 简化Spring应用开发
- 快速构建独立运行的Java应用

**为什么要用它？**
- 开发效率高，不需要写很多配置
- 内置Tomcat服务器，不需要额外安装
- 生态丰富，有很多现成的组件
- 社区活跃，遇到问题容易找到解决方案

**在本项目中的作用：**
- 作为后端框架的基础
- 提供依赖注入、自动配置等功能
- 整合其他技术（MyBatis-Plus、JWT等）

**生活化例子：**
就像盖房子用的脚手架，有了脚手架，盖房子就快多了，不需要从零开始搭建。

**如果不用它会怎样？**
- 需要写很多配置文件
- 开发效率低
- 项目启动慢

---

#### 2. **MyBatis-Plus**

**这是什么？**
- MyBatis的增强工具
- 简化数据库操作
- 自动生成SQL语句

**为什么要用它？**
- 不用手写SQL语句
- 自动完成增删改查操作
- 支持分页、条件查询等常用功能
- 代码量少，开发效率高

**在本项目中的作用：**
- 操作数据库（增删改查）
- 查询快递列表、用户列表等
- 分页查询数据

**生活化例子：**
就像餐厅的点餐系统，你只需要点"宫保鸡丁"，系统自动告诉后厨需要什么材料、怎么做，不需要你亲自去后厨做。

**如果不用它会怎样？**
- 需要手写大量SQL语句
- 代码量大，容易出错
- 开发效率低

---

#### 3. **MySQL 8.0**

**这是什么？**
- 关系型数据库
- 存储和管理数据

**为什么要用它？**
- 开源免费
- 性能好，适合中小型项目
- 社区活跃，资料丰富
- 支持事务，保证数据一致性

**在本项目中的作用：**
- 存储快递信息
- 存储用户信息
- 存储取件记录
- 存储公告信息

**生活化例子：**
就像餐厅的仓库，所有的食材都放在仓库里，需要的时候就去仓库拿。

**如果不用它会怎样？**
- 数据无法持久化保存
- 重启服务器后数据丢失
- 无法进行数据查询和统计

---

#### 4. **JWT（JSON Web Token）**

**这是什么？**
- 一种令牌（Token）机制
- 用于用户身份认证
- 无状态认证方式

**为什么要用它？**
- 不需要在服务器存储会话信息
- 可以跨域使用
- 性能好，减轻服务器压力
- 适合前后端分离架构

**在本项目中的作用：**
- 用户登录后生成JWT令牌
- 前端每次请求都携带令牌
- 后端验证令牌判断用户身份
- 实现权限控制

**生活化例子：**
就像餐厅的会员卡，你办了会员卡后，每次来餐厅出示会员卡，服务员就知道你是谁、有什么优惠，不需要每次都登记。

**如果不用它会怎样？**
- 无法实现用户身份认证
- 任何人都可以访问接口
- 安全性无法保证

---

### 前端技术栈

#### 1. **Vue 3**

**这是什么？**
- JavaScript前端框架
- 用于构建用户界面
- 采用组件化开发

**为什么要用它？**
- 学习曲线平缓，容易上手
- 性能好，运行速度快
- 组件化开发，代码复用性高
- 生态丰富，有很多现成的组件库

**在本项目中的作用：**
- 构建页面
- 处理用户交互
- 展示数据
- 调用后端接口

**生活化例子：**
就像乐高积木，每个页面都是由很多小积木（组件）拼起来的，可以重复使用，拼起来很方便。

**如果不用它会怎样？**
- 需要写大量原生JavaScript代码
- 代码难以维护
- 开发效率低

---

#### 2. **Element Plus**

**这是什么？**
- Vue 3的UI组件库
- 提供了很多现成的UI组件

**为什么要用它？**
- 组件丰富，按钮、表格、表单等都有
- 样式美观，不需要自己写CSS
- 开发效率高，直接用现成的组件
- 文档完善，容易上手

**在本项目中的作用：**
- 提供按钮、表格、表单等UI组件
- 统一页面风格
- 提高开发效率

**生活化例子：**
就像装修用的成品家具，你不需要自己动手做桌子、椅子，直接买现成的就行，而且样式统一好看。

**如果不用它会怎样？**
- 需要自己写很多CSS样式
- 页面风格不统一
- 开发效率低

---

#### 3. **Axios**

**这是什么？**
- HTTP客户端库
- 用于发送HTTP请求

**为什么要用它？**
- 支持Promise，代码简洁
- 支持请求和响应拦截
- 自动转换JSON数据
- 浏览器兼容性好

**在本项目中的作用：**
- 发送HTTP请求到后端
- 接收后端返回的数据
- 统一处理错误

**生活化例子：**
就像餐厅的外卖员，你点餐后，外卖员去餐厅取餐，然后送到你手里。

**如果不用它会怎样？**
- 需要使用原生的XMLHttpRequest或fetch
- 代码复杂，兼容性差
- 错误处理麻烦

---

#### 4. **Vite**

**这是什么？**
- 前端构建工具
- 用于打包和运行前端项目

**为什么要用它？**
- 启动速度快，开发体验好
- 支持热更新，修改代码后立即看到效果
- 打包速度快
- 配置简单

**在本项目中的作用：**
- 运行前端开发服务器
- 打包前端项目
- 处理Vue文件、TypeScript等

**生活化例子：**
就像餐厅的厨房设备，有了这些设备，做菜就快多了，而且可以同时做很多道菜。

**如果不用它会怎样？**
- 开发服务器启动慢
- 修改代码后需要手动刷新页面
- 打包速度慢

## 🏗️ 环境搭建步骤

### 第一步：安装必要软件

#### 1. 安装JDK 17

**为什么需要？**
- Spring Boot 3.x需要JDK 17或更高版本
- Java开发的基础环境

**怎么安装？**
1. 下载JDK 17安装包（Oracle官网或OpenJDK）
2. 运行安装程序，按照提示安装
3. 配置环境变量JAVA_HOME
4. 配置环境变量Path，添加%JAVA_HOME%\bin

**验证是否安装成功：**
打开命令行，输入：
```bash
java -version
```
如果看到类似"java version 17.x.x"的输出，说明安装成功。

---

#### 2. 安装Maven

**为什么需要？**
- 管理Java项目的依赖
- 构建和运行Spring Boot项目

**怎么安装？**
1. 下载Maven安装包（Maven官网）
2. 解压到指定目录（如D:\maven）
3. 配置环境变量MAVEN_HOME
4. 配置环境变量Path，添加%MAVEN_HOME%\bin
5. 配置Maven镜像（加速依赖下载）

**验证是否安装成功：**
打开命令行，输入：
```bash
mvn -version
```
如果看到类似"Apache Maven 3.x.x"的输出，说明安装成功。

---

#### 3. 安装MySQL 8.0**

**为什么需要？**
- 存储项目数据
- 运行数据库服务

**怎么安装？**
1. 下载MySQL 8.0安装包（MySQL官网）
2. 运行安装程序，按照提示安装
3. 设置root用户密码（记住这个密码！）
4. 配置MySQL服务

**验证是否安装成功：**
打开命令行，输入：
```bash
mysql -u root -p
```
输入密码后，如果能进入MySQL命令行，说明安装成功。

---

#### 4. 安装Node.js**

**为什么需要？**
- 运行前端项目
- 安装npm包

**怎么安装？**
1. 下载Node.js安装包（Node.js官网）
2. 运行安装程序，按照提示安装
3. 安装完成后，npm会自动安装

**验证是否安装成功：**
打开命令行，输入：
```bash
node -version
npm -version
```
如果看到版本号输出，说明安装成功。

---

### 第二步：配置数据库

#### 1. 创建数据库

打开MySQL命令行或MySQL Workbench，执行：

```sql
CREATE DATABASE campus_express CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

**解释：**
- `CREATE DATABASE` - 创建数据库
- `campus_express` - 数据库名称
- `CHARACTER SET utf8mb4` - 设置字符集为utf8mb4（支持中文）
- `COLLATE utf8mb4_unicode_ci` - 设置排序规则

---

#### 2. 导入数据库表结构

找到项目中的SQL文件（如果有），执行SQL脚本创建表。

如果没有SQL文件，可以手动创建表（参考entity类中的字段）。

---

### 第三步：配置后端项目

#### 1. 修改数据库配置

打开`src/main/resources/application.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_express?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的MySQL密码
```

**解释：**
- `url` - 数据库连接地址
- `username` - 数据库用户名（通常是root）
- `password` - 数据库密码（你设置的MySQL密码）

---

#### 2. 安装依赖

在项目根目录下打开命令行，执行：

```bash
mvn clean install
```

**解释：**
- `mvn clean` - 清理之前的构建
- `mvn install` - 下载依赖并构建项目

这个过程可能需要几分钟，因为要下载很多依赖包。

---

#### 3. 运行后端项目

在项目根目录下打开命令行，执行：

```bash
mvn spring-boot:run
```

**解释：**
- `mvn spring-boot:run` - 运行Spring Boot项目

如果看到类似"Started CampusExpressApplication in x.xxx seconds"的输出，说明后端启动成功。

后端默认运行在`http://localhost:8080`

---

### 第四步：配置前端项目

#### 1. 安装依赖

在`campus-express-frontend`目录下打开命令行，执行：

```bash
npm install
```

**解释：**
- `npm install` - 安装package.json中定义的依赖

这个过程可能需要几分钟，因为要下载很多npm包。

---

#### 2. 运行前端项目

在`campus-express-frontend`目录下打开命令行，执行：

```bash
npm run dev
```

**解释：**
- `npm run dev` - 运行前端开发服务器

如果看到类似"Local: http://localhost:5173/"的输出，说明前端启动成功。

---

### 第五步：访问系统

打开浏览器，访问：
```
http://localhost:5173/
```

你应该能看到登录页面。

## 🎯 整体观：技术栈在项目中的位置

### 技术栈关系图

```
用户
  ↓ 浏览器
Vue3 + Element Plus（前端界面）
  ↓ HTTP请求
Axios（发送请求）
  ↓
Spring Boot（后端框架）
  ↓
MyBatis-Plus（数据库操作）
  ↓
MySQL（数据库）
```

### 技术选型理由

| 技术 | 作用 | 选型理由 |
|------|------|----------|
| **Spring Boot** | 后端框架 | 开发效率高、生态丰富、社区活跃 |
| **MyBatis-Plus** | 数据库操作 | 简化SQL、代码量少、开发效率高 |
| **MySQL** | 数据库 | 开源免费、性能好、资料丰富 |
| **JWT** | 身份认证 | 无状态、适合前后端分离、性能好 |
| **Vue 3** | 前端框架 | 学习曲线平缓、性能好、组件化 |
| **Element Plus** | UI组件库 | 组件丰富、样式美观、开发效率高 |
| **Axios** | HTTP客户端 | 支持Promise、代码简洁、兼容性好 |
| **Vite** | 构建工具 | 启动快、热更新、打包快 |

## 📌 答辩可用说法示例

### 老师如果问"你为什么选择Spring Boot？"

**你可以这样说：**

> "我选择Spring Boot主要有几个原因：第一，Spring Boot简化了Spring应用的配置，不需要写大量的XML配置文件，开发效率高；第二，Spring Boot内置了Tomcat服务器，不需要额外安装和配置Web服务器；第三，Spring Boot生态丰富，有很多现成的组件和插件，比如我使用的MyBatis-Plus就是基于Spring Boot的；第四，Spring Boot是当前Java开发的主流框架，社区活跃，遇到问题容易找到解决方案。"

### 老师如果问"你为什么选择Vue3而不是React？"

**你可以这样说：**

> "我选择Vue3主要有几个原因：第一，Vue3的学习曲线比较平缓，对于本科毕设来说，学习和开发成本相对较低；第二，Vue3的文档很完善，中文资料丰富，遇到问题容易找到解决方案；第三，Vue3的组件化开发模式很适合构建这种管理系统，代码复用性高；第四，Element Plus是基于Vue3的UI组件库，提供了丰富的现成组件，大大提高了开发效率。当然，React也是一个很好的选择，但综合考虑学习成本和开发效率，我选择了Vue3。"

### 老师如果问"你为什么使用JWT而不是Session？"

**你可以这样说：**

> "我使用JWT主要有几个原因：第一，JWT是无状态的，不需要在服务器存储会话信息，减轻了服务器压力；第二，JWT适合前后端分离的架构，前端只需要在请求头中携带Token，后端验证Token即可；第三，JWT可以跨域使用，方便前端和后端部署在不同的服务器上；第四，JWT的性能比Session好，不需要频繁查询数据库。当然，Session也有它的优势，但对于我这个项目来说，JWT更适合。"

### 老师如果问"你的数据库为什么选择MySQL？"

**你可以这样说：**

> "我选择MySQL主要有几个原因：第一，MySQL是开源免费的，对于学生毕设来说，成本最低；第二，MySQL的性能很好，完全能够满足校园快递管理系统的需求；第三，MySQL的资料非常丰富，遇到问题容易找到解决方案；第四，MySQL支持事务，能够保证数据的一致性，这对于快递管理系统来说非常重要。"

### 老师如果问"如果让你重新选择技术栈，你会怎么选？"

**你可以这样说：**

> "如果让我重新选择，我可能还是会选择当前的技术栈，因为它已经能够很好地满足项目需求了。不过，如果要改进的话，我可能会考虑以下几点：第一，使用Spring Security替代自定义的JWT认证，因为Spring Security更成熟、更安全；第二，使用TypeScript替代JavaScript，因为TypeScript有类型检查，代码更健壮；第三，使用Redis缓存热点数据，提高系统性能。不过，对于本科毕设来说，当前的技术栈已经足够了，不需要过度设计。"

## 💡 学习建议

1. **不要死记硬背技术名称**
   - 理解每个技术的作用
   - 理解为什么选择这个技术
   - 能用自己的话讲出来就行

2. **动手搭建环境**
   - 按照步骤一步步操作
   - 遇到问题不要慌，搜索引擎是你的朋友
   - 成功运行项目会有成就感

3. **理解技术之间的关系**
   - 前端和后端怎么通信
   - 后端和数据库怎么交互
   - 各个技术在整个系统中扮演什么角色

4. **不要追求最新技术**
   - 毕设不是技术竞赛
   - 选择成熟稳定的技术
   - 能够解决问题就行
