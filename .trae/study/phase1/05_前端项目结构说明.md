# 前端项目结构说明

## 🎯 学习目标

通过本章学习，你将能够：
- 理解前端项目的整体目录结构

## 📋 项目概述

前端项目采用**Vue3 + TypeScript + Element Plus**技术栈，使用**Vite**作为构建工具，实现了与后端的完全分离。

### 技术栈组成

| 技术 | 版本 | 作用 |
|------|------|------|
| Vue3 | 3.5.13 | 前端框架，负责页面渲染和交互 |
| TypeScript | 5.7.2 | JavaScript的超集，提供类型安全 |
| Element Plus | 2.9.1 | UI组件库，提供丰富的界面组件 |
| Vite | 6.0.3 | 构建工具，提供快速的开发体验 |
| Vue Router | 4.5.0 | 路由管理，实现页面跳转 |
| Pinia | 2.2.8 | 状态管理，管理全局状态 |
| Axios | 1.13.2 | HTTP客户端，与后端API通信 |
| ECharts | 5.6.0 | 数据可视化库，用于图表展示 |

## 📂 核心目录说明

### 1. src/api/ - API接口层

**作用**：定义所有与后端交互的API接口

**示例**：express.ts
```typescript
import request from '@/utils/request'

export interface Express {
  id?: number
  expressNo: string
  recipientName: string
  recipientPhone: string
  status: number
  pickupCode?: string
  createTime?: string
}

export const getExpressList = (params: any) => {
  return request({
    url: '/api/express/list',
    method: 'get',
    params
  })
}

export const createExpress = (data: Express) => {
  return request({
    url: '/api/express',
    method: 'post',
    data
  })
}
```

**设计要点**：
- 使用TypeScript接口定义数据类型
- 统一使用request工具进行HTTP请求
- 按业务模块划分文件

### 2. src/views/ - 页面组件层

**作用**：实现具体的页面功能

**示例**：express/index.vue
```vue
<template>
  <div class="express-container">
    <el-table :data="expressList">
      <el-table-column prop="expressNo" label="快递单号" />
      <el-table-column prop="recipientName" label="收件人" />
      <el-table-column prop="status" label="状态" />
    </el-table>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { getExpressList } from '@/api/express'
import type { Express } from '@/api/express'

const expressList = ref<Express[]>([])

onMounted(() => {
  loadExpressList()
})

const loadExpressList = async () => {
  const res = await getExpressList({})
  expressList.value = res.data
}
</script>
```

**设计要点**：
- 使用Vue3 Composition API（`<script setup>`）
- TypeScript类型标注
- Element Plus组件库
- 响应式数据管理

### 3. src/router/ - 路由配置

**作用**：管理页面路由和导航

**示例**：router/index.ts
```typescript
import { createRouter, createWebHistory } from 'vue-router'
import MainLayout from '@/layouts/MainLayout.vue'

const routes = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/login/index.vue')
  },
  {
    path: '/',
    component: MainLayout,
    children: [
      {
        path: 'dashboard',
        name: 'Dashboard',
        component: () => import('@/views/dashboard/index.vue')
      },
      {
        path: 'express',
        name: 'Express',
        component: () => import('@/views/express/index.vue')
      }
    ]
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router
```

**设计要点**：
- 使用路由懒加载提升性能
- 嵌套路由结构
- 路由守卫实现权限控制

### 4. src/store/ - 状态管理

**作用**：管理全局状态，如用户信息、权限等

**示例**：store/modules/user.ts
```typescript
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useUserStore = defineStore('user', () => {
  const userInfo = ref<any>(null)
  const token = ref<string>('')

  const setUserInfo = (info: any) => {
    userInfo.value = info
  }

  const setToken = (newToken: string) => {
    token.value = newToken
  }

  return {
    userInfo,
    token,
    setUserInfo,
    setToken
  }
})
```

**设计要点**：
- 使用Pinia进行状态管理
- 模块化组织状态
- 提供getter和action

### 5. src/utils/ - 工具函数

**作用**：提供通用的工具函数

**示例**：utils/request.ts
```typescript
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/store'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 5000
})

request.interceptors.request.use(
  (config) => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = `Bearer ${userStore.token}`
    }
    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)

request.interceptors.response.use(
  (response) => {
    return response.data
  },
  (error) => {
    ElMessage.error(error.message)
    return Promise.reject(error)
  }
)

export default request
```

**设计要点**：
- Axios实例封装
- 请求/响应拦截器
- 统一错误处理
- Token自动注入

## 🔄 数据流转过程

### 1. 用户操作流程

```
用户点击按钮
    ↓
触发Vue组件方法
    ↓
调用API接口函数
    ↓
发送HTTP请求到后端
    ↓
接收后端响应数据
    ↓
更新组件状态
    ↓
页面重新渲染
```

### 2. 具体示例：快递列表加载

```typescript
// 1. 用户进入快递管理页面
<template>
  <el-button @click="loadData">刷新</el-button>
</template>

// 2. 组件调用API
const loadData = async () => {
  const res = await getExpressList({ page: 1, size: 10 })
  expressList.value = res.data
}

// 3. API发送请求
export const getExpressList = (params: any) => {
  return request({
    url: '/api/express/list',
    method: 'get',
    params
  })
}

// 4. 请求拦截器添加Token
request.interceptors.request.use((config) => {
  config.headers.Authorization = `Bearer ${token}`
  return config
})

// 5. 发送到后端
GET http://localhost:8080/api/express/list?page=1&size=10
Headers: Authorization: Bearer xxx

// 6. 后端返回数据
{
  "code": 200,
  "message": "success",
  "data": [...]
}

// 7. 响应拦截器处理
request.interceptors.response.use((response) => {
  return response.data
})

// 8. 组件更新数据
expressList.value = res.data

// 9. 页面重新渲染
<el-table :data="expressList">
```

```typescript
router.beforeEach((to, from, next) => {

  }
})
```

## 📝 学习要点总结

### 核心概念

1. **Vue3 Composition API**
   - 使用`<script setup>`语法
   - `ref`和`reactive`定义响应式数据
   - 生命周期钩子：`onMounted`、`onUpdated`等

2. **TypeScript类型系统**
   - 接口定义数据结构
   - 类型标注提高代码安全性
   - 泛型增强代码复用

3. **前后端分离**
   - 前端负责页面展示和交互
   - 后端提供RESTful API
   - 通过HTTP协议通信

4. **组件化开发**
   - 组件复用提高开发效率
   - Props传递数据
   - Events触发父组件方法

### 毕设答辩要点

1. **为什么选择Vue3？**
   - 性能优化：虚拟DOM diff算法改进
   - Composition API：更好的代码组织
   - TypeScript支持：类型安全

2. **前后端分离的优势？**
   - 职责分离，开发并行
   - 接口统一，易于维护
   - 技术栈独立，灵活选择

3. **如何处理跨域问题？**
   - 开发环境：Vite代理配置
   - 生产环境：Nginx反向代理

4. **状态管理的作用？**
   - 共享用户信息
   - 全局状态同步
   - 组件间通信简化

### 常见问题

**Q1: 为什么要用TypeScript？**
A: TypeScript提供静态类型检查，在编译时就能发现错误，提高代码质量和可维护性。

**Q2: Vite相比Webpack有什么优势？**
A: Vite使用ES Module，开发服务器启动速度快，热更新效率高。

**Q3: Element Plus是什么？**
A: Element Plus是基于Vue3的UI组件库，提供了丰富的预置组件，快速构建美观的界面。

**Q4: 如何优化前端性能？**
A: 路由懒加载、组件按需引入、图片压缩、代码分割等。
