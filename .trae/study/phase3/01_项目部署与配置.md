# 项目部署与配置

## 🎯 学习目标

通过本章学习，你将能够：
- 掌握后端项目的部署流程
- 掌握前端项目的构建与部署
- 了解生产环境的配置优化
- 掌握数据库的备份与恢复

## 📋 部署概述

本项目采用**前后端分离架构**，需要分别部署：
- **后端**：Spring Boot应用，部署在Tomcat或直接运行JAR包
- **前端**：Vue3静态资源，部署在Nginx
- **数据库**：MySQL数据库

## 🏗️ 部署架构图

```
用户浏览器
    ↓
Nginx (反向代理 + 静态资源)
    ├── / → 前端静态文件
    └── /api → 后端Spring Boot应用
        ↓
    MySQL数据库
```

## 🔧 后端部署

### 1. 环境准备

#### 1.1 安装JDK 17

```bash
# 检查Java版本
java -version

# 如果版本不对，安装JDK 17
# Windows: 下载并安装Oracle JDK 17或OpenJDK 17
# Linux:
sudo apt update
sudo apt install openjdk-17-jdk
```

#### 1.2 安装MySQL 8

```bash
# Windows: 下载并安装MySQL 8.0
# Linux:
sudo apt update
sudo apt install mysql-server

# 启动MySQL服务
sudo systemctl start mysql
sudo systemctl enable mysql
```

#### 1.3 创建数据库

```sql
-- 登录MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE campus_express CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户并授权
CREATE USER 'campus'@'localhost' IDENTIFIED BY 'Campus@2024';
GRANT ALL PRIVILEGES ON campus_express.* TO 'campus'@'localhost';
FLUSH PRIVILEGES;
```

### 2. 配置生产环境

#### 2.1 修改application-prod.yml

```yaml
server:
  port: 8080

    url: jdbc:mysql://localhost:3306/campus_express?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: campus

    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000

  level:
```

#### 2.2 修改pom.xml配置

```xml

    <plugins>
```

### 3. 构建项目

```bash
# 进入后端项目目录
cd src

# 清理并打包
mvn clean package -DskipTests

# 生成的JAR包位置
# target/campus-express-1.0.0.jar
```

### 4. 运行项目

#### 4.1 直接运行JAR包

```bash
# 前台运行
java -jar target/campus-express-1.0.0.jar --spring.profiles.active=prod

# 后台运行（Linux）
nohup java -jar target/campus-express-1.0.0.jar --spring.profiles.active=prod > /dev/null 2>&1 &

# 查看日志
tail -f /var/log/campus-express/application.log
```

#### 4.2 使用systemd管理服务（Linux）

创建服务文件：`/etc/systemd/system/campus-express.service`

```ini
[Unit]
Description=Campus Express Backend Service
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/campus-express
ExecStart=/usr/bin/java -jar /opt/campus-express/campus-express-1.0.0.jar --spring.profiles.active=prod
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载systemd配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start campus-express

# 设置开机自启
sudo systemctl enable campus-express

# 查看服务状态
sudo systemctl status campus-express

# 查看日志
sudo journalctl -u campus-express -f
```

### 5. 验证部署

```bash
# 检查端口是否监听
netstat -tlnp | grep 8080

# 测试API
curl http://localhost:8080/api/health
```

### 4. 部署到Nginx

#### 4.1 安装Nginx

```bash
# Linux
sudo apt update
sudo apt install nginx

# 启动Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

#### 4.2 配置Nginx

创建配置文件：`/etc/nginx/sites-available/campus-express`

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态资源
    location / {
        root /var/www/campus-express/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/campus-express/dist;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/javascript application/json;
}
```

#### 4.3 启用配置

```bash
# 创建符号链接
sudo ln -s /etc/nginx/sites-available/campus-express /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

## 🗄️ 数据库部署

### 1. 数据库备份

#### 1.1 手动备份

```bash
# 备份数据库
mysqldump -u campus -p campus_express > backup_$(date +%Y%m%d_%H%M%S).sql

# 压缩备份
gzip backup_$(date +%Y%m%d_%H%M%S).sql
```

#### 1.2 自动备份脚本

创建脚本：`/opt/scripts/backup.sh`

```bash
#!/bin/bash

# 配置
DB_USER="campus"
DB_PASS="Campus@2024"
DB_NAME="campus_express"
BACKUP_DIR="/var/backups/campus-express"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u $DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/backup_$DATE.sql.gz

# 删除7天前的备份
find $BACKUP_DIR -name "backup_*.sql.gz" -mtime +7 -delete

echo "Backup completed: backup_$DATE.sql.gz"
```

设置定时任务：

```bash
# 编辑crontab
crontab -e

# 添加定时任务（每天凌晨2点备份）
0 2 * * * /opt/scripts/backup.sh
```

### 2. 数据库恢复

```bash
# 解压备份文件
gunzip backup_20240114_020000.sql.gz

# 恢复数据库
mysql -u campus -p campus_express < backup_20240114_020000.sql
```

## 🚀 Docker部署（可选）

### 1. 后端Dockerfile

创建文件：`Dockerfile`

```dockerfile
FROM openjdk:17-jdk-slim

WORKDIR /app

COPY target/campus-express-1.0.0.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar", "--spring.profiles.active=prod"]
```

### 2. 前端Dockerfile

创建文件：`Dockerfile`

```dockerfile
FROM node:18-alpine as builder

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html

COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 3. docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: campus-mysql
    environment:
      MYSQL_ROOT_PASSWORD: Root@2024
      MYSQL_DATABASE: campus_express
      MYSQL_USER: campus
      MYSQL_PASSWORD: Campus@2024
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - campus-network

  backend:
    build: ./src
    container_name: campus-backend
    ports:
      - "8080:8080"
    depends_on:
      - mysql
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/campus_express?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: campus
      SPRING_DATASOURCE_PASSWORD: Campus@2024
    networks:
      - campus-network

  frontend:
    build: ./campus-express-frontend
    container_name: campus-frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - campus-network

volumes:
  mysql-data:

networks:
  campus-network:
    driver: bridge
```

### 4. 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 🔍 监控与日志

### 1. 应用监控

#### 1.1 Spring Boot Actuator

在`pom.xml`中添加依赖：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

配置端点：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics
  endpoint:
    health:
      show-details: always
```

访问监控端点：
- http://localhost:8080/actuator/health
- http://localhost:8080/actuator/metrics

### 2. 日志管理

#### 2.1 Logback配置

创建文件：`src/main/resources/logback-spring.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <property name="LOG_PATH" value="/var/log/campus-express"/>
    <property name="APP_NAME" value="campus-express"/>

    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/${APP_NAME}.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_PATH}/${APP_NAME}-%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <timeBasedFileNamingAndTriggeringPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedFNATP">
                <maxFileSize>100MB</maxFileSize>
            </timeBasedFileNamingAndTriggeringPolicy>
            <maxHistory>30</maxHistory>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>

    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="FILE"/>
    </root>
</configuration>
```

## 📝 学习要点总结

### 核心概念

1. **前后端分离部署**
   - 后端：Spring Boot JAR包
   - 前端：Nginx静态资源
   - 通信：HTTP API

2. **生产环境配置**
   - 数据库连接池优化
   - 日志配置
   - JVM参数调优

3. **反向代理**
   - Nginx作为反向代理
   - 静态资源服务
   - API请求转发

4. **数据库备份**
   - 定时备份策略
   - 压缩存储
   - 自动清理过期备份

### 毕设答辩要点

1. **为什么选择前后端分离部署？**
   - 职责分离，便于维护
   - 前端可以独立部署
   - 后端可以水平扩展

2. **如何保证数据安全？**
   - HTTPS加密传输
   - 数据库定期备份
   - 敏感信息配置加密

3. **如何优化系统性能？**
   - 前端资源压缩和缓存
   - 后端连接池优化
   - 数据库索引优化

4. **如何监控系统运行状态？**
   - Spring Boot Actuator
   - 日志分析
   - 性能监控工具

### 常见问题

**Q1: 部署后无法访问？**
A: 检查防火墙设置、端口监听状态、Nginx配置是否正确。

**Q2: 数据库连接失败？**
A: 检查数据库服务是否启动、连接配置是否正确、网络是否通畅。

**Q3: 前端页面空白？**
A: 检查构建是否成功、Nginx静态文件路径是否正确、浏览器控制台是否有错误。

**Q4: 如何实现自动部署？**
A: 使用CI/CD工具如Jenkins、GitLab CI等，实现代码提交后自动构建和部署。
