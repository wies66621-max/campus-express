# 毕设答辩准备

## 🎯 学习目标

通过本章学习，你将能够：
- 掌握毕设答辩的准备工作
- 了解答辩流程和注意事项
- 学会回答常见答辩问题
- 掌握演示技巧和注意事项

## 📋 答辩概述

毕业设计答辩是检验学生学习成果的重要环节，主要包括：
1. **PPT演示**：展示项目成果和技术亮点
2. **系统演示**：实际操作展示系统功能
3. **问题回答**：回答评委老师提出的问题
4. **技术交流**：与评委老师进行技术讨论

#### 1.3 项目背景与意义（2-3页）

**内容要点**：

1. **研究背景**
   - 校园快递业务快速增长
   - 传统管理方式效率低下
   - 信息化管理需求迫切

2. **研究意义**
   - 提高快递管理效率
   - 方便师生取件寄件
   - 降低管理成本
   - 提升服务质量

3. **国内外现状**
   - 国内：顺丰、菜鸟等快递公司信息化程度高
   - 国外：亚马逊、FedEx等先进管理系统
   - 校园：信息化程度相对较低，有改进空间

**PPT示例**：

```
项目背景

随着电子商务的快速发展，校园快递业务量急剧增长：
• 2023年全国快递业务量突破1100亿件
• 校园快递日均处理量达数千件
• 传统手工管理方式效率低下，易出错

存在问题：
✗ 快递信息记录混乱
✗ 取件流程繁琐
✗ 数据统计困难
✗ 无法实时监控
```

#### 1.4 需求分析与系统设计（3-4页）

**内容要点**：

1. **功能需求**
   - 用户管理：注册、登录、权限控制
   - 快递管理：入库、出库、查询、统计
   - 站点管理：站点信息、快递员管理
   - 取件管理：取件码生成、验证
   - 数据统计：报表、图表展示

2. **非功能需求**
   - 性能：响应时间<2秒
   - 安全：数据加密、权限控制
   - 可用性：7×24小时运行
   - 可扩展性：支持多站点部署

3. **系统架构**
   - 前后端分离架构
   - 三层架构：表现层、业务层、数据层
   - RESTful API设计

**PPT示例**：


## 🗣️ 答辩问题准备

### 1. 常见问题分类

#### 1.1 项目背景类

**Q1: 为什么选择这个课题？**

参考答案：
"随着电子商务的快速发展，校园快递业务量急剧增长。传统的手工管理方式效率低下，容易出错，无法满足现代校园快递管理的需求。因此，我选择开发校园快递管理系统，希望通过信息化手段提高管理效率，方便师生取件寄件。"

**Q2: 这个项目的创新点在哪里？**

参考答案：
"本项目的创新点主要体现在以下几个方面：
1. 取件码生成算法：采用随机数+时间戳+校验位的方式，保证取件码的唯一性和安全性
2. 数据可视化：使用ECharts实现数据统计和图表展示，直观展示快递业务情况
3. 前后端分离：采用Vue3+Spring Boot前后端分离架构，提高开发效率和系统可维护性"

#### 1.2 技术实现类

**Q3: 为什么选择Spring Boot作为后端框架？**

参考答案：
"选择Spring Boot主要有以下几个原因：
1. 成熟稳定：Spring Boot是Java生态中最流行的框架之一，社区活跃，文档完善
2. 开发效率高：自动配置、起步依赖等特性大大简化了开发工作
3. 生态完善：集成了大量的第三方库，如MyBatis-Plus、Spring Security等
4. 部署方便：可以直接打包成JAR包运行，部署简单"

**Q4: 为什么选择Vue3作为前端框架？**

参考答案：
"选择Vue3主要有以下几个原因：
1. 轻量高效：Vue3体积小，性能优秀，适合中小型项目
2. 易于学习：Vue3的语法简洁，学习曲线平缓
3. Composition API：提供了更好的代码组织方式，便于维护
4. 生态完善：Element Plus等UI组件库提供了丰富的组件，快速开发"

**Q5: 如何保证系统的安全性？**

参考答案：
"系统安全性主要从以下几个方面保证：
1. 身份认证：使用JWT进行无状态认证，避免Session管理
2. 权限控制：基于角色的访问控制（RBAC），不同角色有不同的权限
3. 数据加密：敏感数据（如密码）使用BCrypt加密存储
4. SQL注入防护：使用MyBatis-Plus的参数化查询，避免SQL注入
5. XSS防护：前端对用户输入进行过滤，防止XSS攻击"

#### 1.3 系统设计类

**Q6: 取件码是如何生成的？如何保证唯一性？**

参考答案：
"取件码生成采用以下算法：
1. 生成6位随机数字
2. 加上时间戳的后4位
3. 计算校验位
4. 组合成最终的取件码

唯一性保证：
1. 数据库中设置唯一索引
2. 生成时查询数据库，如果重复则重新生成
3. 使用时间戳保证不同时间生成的取件码不同"

**Q7: 系统的数据库是如何设计的？**

参考答案：
"数据库设计遵循第三范式，主要包含以下表：
1. 用户表（user）：存储用户信息
2. 快递表（express）：存储快递信息
3. 站点表（station）：存储站点信息
4. 取件记录表（pickup_record）：存储取件记录

设计原则：
1. 合理设置主键和外键
2. 为常用查询字段添加索引
3. 使用合适的字段类型和长度
4. 设置字段约束（非空、唯一等）"

#### 1.4 性能优化类

**Q8: 如何优化系统性能？**

参考答案：
"系统性能优化主要从以下几个方面进行：
1. 数据库优化：
   - 为常用查询字段添加索引
   - 使用分页查询，避免全表扫描
   - 优化SQL语句，避免复杂查询
2. 前端优化：
   - 使用路由懒加载
   - 启用Gzip压缩
   - 配置静态资源缓存
3. 后端优化：
   - 使用连接池管理数据库连接
   - 使用缓存减少数据库查询
   - 异步处理耗时操作"

#### 1.5 项目难点类

**Q9: 开发过程中遇到的最大困难是什么？如何解决的？**

参考答案：
"开发过程中遇到的最大困难是前后端数据交互的问题。最初在处理跨域问题时遇到了很多困难，后来通过以下方式解决：
1. 后端配置CORS，允许前端跨域访问
2. 前端使用Axios拦截器统一处理请求和响应
3. 使用TypeScript定义接口类型，保证数据类型一致

通过这次经历，我学到了很多关于前后端交互的知识，也提高了问题解决能力。"

| 创新性 | 15% | 创新点、技术亮点 |

   - 系统是否稳定

2. **技术水平**
   - 技术选型是否合理
   - 代码质量是否高
   - 架构设计是否合理

3. **创新性**
   - 是否有创新点
