## 🔧 环境配置问题

### 问题1：JDK版本不匹配

**现象**：
```
Unsupported class file major version 61
```

**原因**：项目使用JDK 17编译，但运行环境使用的是低版本JDK

**解决方案**：

1. 检查当前JDK版本
```bash
java -version
```

2. 安装JDK 17
```bash
# Windows: 下载并安装Oracle JDK 17
# Linux:
sudo apt install openjdk-17-jdk
```

3. 配置环境变量
```bash
# Linux编辑 ~/.bashrc
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH

# 重新加载配置
source ~/.bashrc
```

### 问题2：Maven依赖下载失败

**现象**：
```
Could not resolve dependencies for project
```

**原因**：Maven仓库访问失败或网络问题

**解决方案**：

1. 配置国内镜像源
```xml
<!-- 在 ~/.m2/settings.xml 中配置 -->
<mirrors>
    <mirror>
        <id>aliyun</id>
        <mirrorOf>central</mirrorOf>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```

2. 清理并重新下载
```bash
mvn clean install -U
```

### 问题3：Node.js版本过低

**现象**：
```
The engine "node" is incompatible with this module
```

**原因**：项目要求Node.js 18+，但当前版本过低

**解决方案**：

1. 使用nvm管理Node.js版本
```bash
# 安装nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# 安装Node.js 18
nvm install 18
nvm use 18
```

2. 或直接下载安装Node.js 18+

## 💻 后端问题

**现象**：
```
Communications link failure
```

**原因**：MySQL服务未启动或连接配置错误

**解决方案**：

1. 检查MySQL服务状态

```bash
sudo systemctl status mysql
```

2. 检查连接配置

```bash
mysql -u campus -p campus_express
```

### 问题5：MyBatis-Plus查询报错

**现象**：
```
Invalid bound statement (not found)
```

**原因**：Mapper接口和XML文件不匹配或扫描路径错误

**解决方案**：

1. 检查Mapper接口位置
```java
@Mapper
public interface ExpressMapper extends BaseMapper<Express> {
}
```

2. 检查XML文件位置
```xml
<!-- src/main/resources/mapper/ExpressMapper.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" 
    "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.school.campusexpress.mapper.ExpressMapper">
</mapper>
```

3. 配置扫描路径
```yaml
mybatis-plus:
  mapper-locations: classpath:mapper/*.xml
```

### 问题6：跨域问题

**现象**：
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**原因**：前后端分离，端口不同导致跨域

**解决方案**：

1. 后端配置CORS
```java
@Configuration
public class CorsConfig {
    @Bean
    public CorsFilter corsFilter() {
        CorsConfiguration config = new CorsConfiguration();
        config.addAllowedOrigin("*");
        config.addAllowedMethod("*");
        config.addAllowedHeader("*");
        config.setAllowCredentials(true);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", config);
        
        return new CorsFilter(source);
    }
}
```

2. 或使用注解
```java
@CrossOrigin(origins = "*", maxAge = 3600)
@RestController
@RequestMapping("/api/express")
public class ExpressController {
}
```

### 问题7：JWT Token验证失败

**现象**：
```
JWT strings must contain exactly 2 period characters
```

**原因**：Token格式错误或过期

**解决方案**：

1. 检查Token生成逻辑
```java
public String generateToken(String username) {
    Date now = new Date();
    Date expiryDate = new Date(now.getTime() + jwtExpirationMs);
    
    return Jwts.builder()
            .setSubject(username)
            .setIssuedAt(now)
            .setExpiration(expiryDate)
            .signWith(SignatureAlgorithm.HS512, jwtSecret)
            .compact();
}
```

2. 检查Token验证逻辑
```java
public String getUsernameFromToken(String token) {
    return Jwts.parser()
            .setSigningKey(jwtSecret)
            .parseClaimsJws(token)
            .getBody()
            .getSubject();
}
```

3. 前端正确存储Token
```typescript
// 存储Token
localStorage.setItem('token', token)

// 请求时携带Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

## 🎨 前端问题

**现象**：
```

```

**原因**：API路径错误或后端服务未启动

**解决方案**：

1. 检查API路径

```typescript
// 错误示例
const res = await getExpressList({ page: 1, size: 10 })

const res = await getExpressList({ page: 1, size: 10 })
```

```bash
curl http://localhost:8080/api/express/list
```

3. 检查Vite代理配置
```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

### 问题9：Element Plus组件样式丢失

**现象**：组件显示异常，样式不正确

**原因**：未正确引入Element Plus样式

**解决方案**：

1. 在main.ts中引入样式
```typescript
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'

const app = createApp(App)
app.use(ElementPlus)
app.mount('#app')
```

2. 按需引入
```typescript
import { ElButton, ElTable } from 'element-plus'
import 'element-plus/theme-chalk/el-button.css'
import 'element-plus/theme-chalk/el-table.css'

app.use(ElButton)
app.use(ElTable)
```

### 问题10：Vue3响应式数据不更新

**现象**：修改数据后页面没有更新

**原因**：响应式数据定义方式错误

**解决方案**：

1. 使用ref定义基本类型
```typescript
// 错误示例
let count = 0

// 正确示例
import { ref } from 'vue'
const count = ref(0)

// 修改值
count.value = 1
```

2. 使用reactive定义对象
```typescript
import { reactive } from 'vue'
const user = reactive({
  name: '张三',
  age: 20
})

// 修改值
user.name = '李四'
```

3. 使用toRefs解构
```typescript
import { reactive, toRefs } from 'vue'
const user = reactive({
  name: '张三',
  age: 20
})

const { name, age } = toRefs(user)
```

### 问题11：路由跳转后页面空白

**现象**：点击导航后页面显示空白

**原因**：路由配置错误或组件路径错误

**解决方案**：

1. 检查路由配置
```typescript
const routes = [
  {
    path: '/express',
    name: 'Express',
    component: () => import('@/views/express/index.vue')
  }
]
```

2. 检查组件路径是否正确
```typescript
// 错误示例
component: () => import('@/views/express.vue')

// 正确示例
component: () => import('@/views/express/index.vue')
```

3. 检查是否添加了router-view
```vue
<template>
  <div id="app">
    <router-view />
  </div>
</template>
```

## 🚀 部署问题

### 问题12：Nginx 502错误

**现象**：
```
502 Bad Gateway
```

**原因**：后端服务未启动或端口配置错误

**解决方案**：

1. 检查后端服务状态
```bash
sudo systemctl status campus-express
```

2. 检查Nginx配置
```nginx
location /api {
    proxy_pass http://localhost:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

3. 检查端口监听
```bash
netstat -tlnp | grep 8080
```

### 问题13：静态资源404

**现象**：页面加载后CSS、JS文件404

**原因**：Nginx静态文件路径配置错误

**解决方案**：

1. 检查静态文件路径
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /var/www/campus-express/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

2. 检查文件是否存在
```bash
ls -la /var/www/campus-express/dist/
```

3. 检查文件权限
```bash
sudo chown -R www-data:www-data /var/www/campus-express
sudo chmod -R 755 /var/www/campus-express
```

### 问题14：数据库连接超时

**现象**：
```
Communications link failure
The last packet sent successfully to the server was 0 milliseconds ago
```

**原因**：数据库连接池配置不当或网络问题

**解决方案**：

1. 调整连接池配置
```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
```

2. 检查数据库服务器性能
```sql
SHOW PROCESSLIST;
SHOW VARIABLES LIKE 'max_connections';
```

3. 优化数据库配置
```ini
[mysqld]
max_connections = 200
wait_timeout = 28800
interactive_timeout = 28800
```

## ⚡ 性能问题

### 问题15：页面加载慢

**现象**：首次访问页面加载时间过长

**原因**：资源未优化或网络问题

**解决方案**：

1. 启用Gzip压缩
```nginx
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/javascript application/json;
```

2. 配置静态资源缓存
```nginx
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

3. 使用CDN加速
```html
<script src="https://cdn.jsdelivr.net/npm/vue@3.5.13/dist/vue.global.prod.js"></script>
```

### 问题16：数据库查询慢

**现象**：API响应时间长

**原因**：缺少索引或SQL语句不优化

**解决方案**：

1. 添加索引
```sql
-- 为常用查询字段添加索引
CREATE INDEX idx_express_no ON express(express_no);
CREATE INDEX idx_recipient_phone ON express(recipient_phone);
CREATE INDEX idx_status ON express(status);
CREATE INDEX idx_create_time ON express(create_time);
```

2. 优化SQL查询
```java
// 使用索引字段查询
LambdaQueryWrapper<Express> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Express::getExpressNo, expressNo);
Express express = expressMapper.selectOne(wrapper);

// 避免全表扫描
// 错误示例
List<Express> list = expressMapper.selectList(null);

// 正确示例
LambdaQueryWrapper<Express> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Express::getStatus, 0);
List<Express> list = expressMapper.selectList(wrapper);
```

3. 使用分页查询
```java
Page<Express> page = new Page<>(pageNum, pageSize);
Page<Express> result = expressMapper.selectPage(page, wrapper);
```

### 问题17：内存占用过高

**现象**：应用运行一段时间后内存占用持续增长

**原因**：内存泄漏或配置不当

**解决方案**：

1. 调整JVM参数
```bash
java -jar \
  -Xms512m \
  -Xmx1024m \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  campus-express-1.0.0.jar
```

2. 分析内存使用
```bash
# 生成堆转储文件
jmap -dump:format=b,file=heapdump.hprof <pid>

# 使用MAT分析堆转储文件
```

3. 检查代码中的内存泄漏
```java
// 避免静态集合无限增长
// 错误示例
public static List<Express> cache = new ArrayList<>();

// 正确示例
public static Map<String, Express> cache = new LinkedHashMap<String, Express>() {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, Express> eldest) {
        return size() > 1000;
    }
};
```
