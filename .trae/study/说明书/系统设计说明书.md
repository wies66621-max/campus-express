# 校园快递管理系统 - 系统设计说明书

## 一、系统概述

### 1.1 项目背景
随着校园快递业务的快速增长，传统的快递管理方式已无法满足需求。本系统旨在为校园提供一套高效、便捷的快递管理解决方案，实现快递的录入、查询、取件等全流程管理。

### 1.2 系统目标
- 提高快递管理效率，减少人工操作
- 实现快递信息的数字化管理
- 为师生提供便捷的快递查询和取件服务
- 支持管理员对快递的统一管理

### 1.3 技术架构
- **后端技术**：Java 17 + Spring Boot 3.1.5 + MyBatis-Plus 3.5.5
- **数据库**：MySQL 8.0
- **前端技术**：Vue 3 + Element Plus
- **接口规范**：RESTful API
- **认证方式**：JWT Token

## 二、系统架构设计

### 2.1 整体架构
系统采用前后端分离的架构设计，分为表现层、业务逻辑层、数据访问层和数据库层。

```
┌─────────────────────────────────────┐
│         前端表现层 (Vue 3)          │
│  - 用户界面                          │
│  - 管理员界面                        │
└──────────────┬──────────────────────┘
               │ HTTP/HTTPS
┌──────────────▼──────────────────────┐
│         控制层 (Controller)          │
│  - AdminExpressController           │
│  - UserExpressController            │
│  - UserController                    │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│       业务逻辑层 (Service)           │
│  - ExpressService                   │
│  - UserService                      │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│       数据访问层 (Mapper)            │
│  - ExpressMapper                    │
│  - UserMapper                       │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         数据库层 (MySQL)            │
│  - express 表                       │
│  - user 表                          │
└─────────────────────────────────────┘
```

### 2.2 分层设计说明

#### 2.2.1 控制层 (Controller)
负责接收前端请求，进行参数校验，调用业务逻辑层处理，并返回统一格式的响应结果。

**主要控制器：**
- `AdminExpressController`：管理员快递管理接口
- `UserExpressController`：用户快递查询接口
- `UserController`：用户认证和管理接口

#### 2.2.2 业务逻辑层 (Service)
实现系统的核心业务逻辑，包括快递的增删改查、取件码生成、状态管理等。

**主要服务：**
- `ExpressService`：快递管理业务逻辑
- `UserService`：用户管理业务逻辑

#### 2.2.3 数据访问层 (Mapper)
使用 MyBatis-Plus 提供的 CRUD 接口，简化数据库操作。

**主要 Mapper：**
- `ExpressMapper`：快递数据访问
- `UserMapper`：用户数据访问

## 三、功能模块设计

### 3.1 用户管理模块

#### 3.1.1 功能描述
实现用户注册、登录、信息管理等功能，支持管理员和普通用户两种角色。

#### 3.1.2 主要功能
- 用户注册：支持学生和管理员注册
- 用户登录：基于 JWT Token 的身份认证
- 用户信息查询：查询用户基本信息
- 用户信息修改：修改用户个人信息

#### 3.1.3 接口设计
```
POST /api/user/register  - 用户注册
POST /api/user/login     - 用户登录
GET  /api/user/info      - 获取用户信息
PUT  /api/user/update    - 更新用户信息
```

### 3.2 快递管理模块

#### 3.2.1 功能描述
实现快递的录入、查询、更新、删除等功能，支持管理员和用户两种角色的不同操作权限。

#### 3.2.2 管理员功能

**快递录入**
- 输入快递单号、收件人姓名、收件人电话、快递站ID等信息
- 系统自动生成6位取件码
- 自动记录入库时间和操作管理员ID

**快递查询**
- 按快递站ID查询快递列表
- 按快递状态查询（待取件、已取件、超期）
- 支持分页查询

**快递管理**
- 查看快递详情
- 更新快递信息
- 删除快递记录

#### 3.2.3 用户功能

**取件码取件**
- 输入6位取件码
- 系统验证取件码有效性
- 更新快递状态为"已取件"
- 记录取件时间

**快递查询**
- 按快递单号搜索
- 按收件人手机号搜索
- 查看我的快递列表

#### 3.2.4 接口设计

**管理员接口**
```
POST   /api/admin/express/add          - 录入快递
GET    /api/admin/express/list         - 快递列表查询
GET    /api/admin/express/{id}         - 快递详情查询
PUT    /api/admin/express/update       - 更新快递信息
DELETE /api/admin/express/delete/{id}  - 删除快递
```

**用户接口**
```
POST /api/user/express/pickup     - 取件码取件
GET  /api/user/express/my         - 我的快递
GET  /api/user/express/search/no  - 按单号搜索
GET  /api/user/express/search/phone - 按手机号搜索
```

## 四、数据库设计

### 4.1 数据库表结构

## 五、关键技术实现

### 5.1 JWT 认证机制

#### 5.1.1 认证流程
1. 用户登录时，验证用户名和密码
2. 验证成功后，生成 JWT Token
3. Token 包含用户ID、角色等信息
4. 客户端在后续请求中携带 Token
5. 服务端验证 Token 有效性

#### 5.1.2 Token 结构
```
Header.Payload.Signature
```

### 5.2 密码加密

#### 5.2.1 加密方式
使用 Hutool 工具库的 MD5 + 盐值加密方式：
- 生成随机盐值
- 密码 + 盐值进行 MD5 加密
- 存储格式：盐值 + 加密结果

#### 5.2.2 验证流程
1. 提取存储的盐值
2. 输入密码 + 盐值进行 MD5 加密
3. 对比加密结果与存储值

### 5.3 取件码生成

#### 5.3.1 生成规则
- 生成6位数字随机码
- 检查数据库中是否已存在
- 如果存在则重新生成，最多重试10次

#### 5.3.2 生成工具类
```java
public class PickupCodeUtil {
    public static String generate() {
        return String.format("%06d", random.nextInt(1000000));
    }
}
```

### 5.4 统一响应格式

#### 5.4.1 响应结构
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

#### 5.4.2 状态码定义
- 200：操作成功
- 401：未授权
- 403：禁止访问
- 500：服务器错误

## 六、系统安全设计

### 6.1 身份认证
- 基于 JWT Token 的无状态认证
- Token 有效期控制
- 密码加密存储

### 6.2 权限控制
- 基于角色的访问控制（RBAC）
- 管理员和用户权限分离
- 接口级别的权限验证

### 6.3 数据安全
- 敏感信息加密存储
- SQL 注入防护（MyBatis-Plus 预编译）
- XSS 攻击防护

## 八、系统部署

### 8.1 部署步骤
1. 安装 Java 17 和 MySQL 8.0
2. 导入数据库脚本 `init.sql`
3. 修改 `application.yml` 配置文件
4. 使用 Maven 打包项目：`mvn clean package`
5. 运行 JAR 文件：`java -jar campus-express-0.0.1-SNAPSHOT.jar`

### 8.2 配置说明
```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_express
    username: root
    password: your_password

mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```
