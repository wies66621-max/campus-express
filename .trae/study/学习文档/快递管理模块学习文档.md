# 快递管理模块学习文档

## 一、模块概述

### 1.1 功能介绍
快递管理模块是校园快递管理系统的核心模块，负责快递信息的录入、查询、更新和取件等核心业务功能。该模块分为管理员端和用户端两个部分：

**管理员端功能：**
- 快递信息录入
- 快递列表查询
- 快递详情查看
- 快递信息更新
- 快递信息删除

**用户端功能：**
- 按取件码取件
- 按手机号查询快递
- 按快递单号查询快递
- 查看我的快递

### 1.2 技术栈
- **后端框架**：Spring Boot 3.1.5
- **ORM框架**：MyBatis-Plus 3.5.5
- **数据库**：MySQL 8.0
- **Java版本**：Java 17
- **接口文档**：Swagger 3

### 1.3 模块架构
采用经典的MVC三层架构：
- **Controller层**：接收HTTP请求，处理参数，返回响应
- **Service层**：业务逻辑处理，数据校验，事务控制
- **Mapper层**：数据库操作，SQL执行

## 二、数据库设计

### 2.1 表结构

```sql
CREATE TABLE `express` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '快递ID',
  `tracking_number` varchar(50) NOT NULL COMMENT '快递单号',
  `company` varchar(50) DEFAULT NULL COMMENT '快递公司',
  `receiver_name` varchar(50) NOT NULL COMMENT '收件人姓名',
  `receiver_phone` varchar(20) NOT NULL COMMENT '收件人电话',
  `pickup_code` varchar(10) DEFAULT NULL COMMENT '取件码',
  `station_id` bigint NOT NULL COMMENT '快递站ID',
  `status` tinyint DEFAULT '0' COMMENT '快递状态(0-已入库,1-待取件,2-已取件)',
  `remark` varchar(255) DEFAULT NULL COMMENT '备注',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `tracking_number` (`tracking_number`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='快递信息表';
```

### 2.2 字段说明

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | bigint | 快递ID，主键 | 自增，非空 |
| tracking_number | varchar(50) | 快递单号 | 唯一，非空 |
| company | varchar(50) | 快递公司 | 可空 |
| receiver_name | varchar(50) | 收件人姓名 | 非空 |
| receiver_phone | varchar(20) | 收件人电话 | 非空 |
| pickup_code | varchar(10) | 取件码 | 可空，系统自动生成 |
| station_id | bigint | 快递站ID | 非空 |
| status | tinyint | 快递状态 | 默认0 |
| remark | varchar(255) | 备注 | 可空 |
| create_time | datetime | 创建时间 | 自动生成 |
| update_time | datetime | 更新时间 | 自动更新 |

### 2.3 状态说明
- **0**：已入库（快递刚录入系统）
- **1**：待取件（已通知用户，等待取件）
- **2**：已取件（用户已完成取件）

### 5.2 核心业务逻辑实现

#### 5.2.1 添加快递

```java
@Override
public Express addExpress(Express express) {
    if (express.getTrackingNumber() == null || express.getTrackingNumber().trim().isEmpty()) {
        throw new RuntimeException("快递单号不能为空");
    }

    if (express.getReceiverName() == null || express.getReceiverName().trim().isEmpty()) {
        throw new RuntimeException("收件人姓名不能为空");
    }

    if (express.getReceiverPhone() == null || express.getReceiverPhone().trim().isEmpty()) {
        throw new RuntimeException("收件人电话不能为空");
    }

    LambdaQueryWrapper<Express> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Express::getTrackingNumber, express.getTrackingNumber());
    Express existExpress = getOne(wrapper);
    if (existExpress != null) {
        throw new RuntimeException("快递单号已存在");
    }

    String pickupCode = PickupCodeUtil.generatePickupCode();
    express.setPickupCode(pickupCode);
    express.setStatus(0);

    save(express);
    return express;
}
```

**业务逻辑说明：**
1. **参数校验**：检查必填字段是否为空
2. **重复性检查**：确保快递单号唯一
3. **取件码生成**：自动生成6位取件码
4. **状态设置**：默认设置为"已入库"状态
5. **数据保存**：调用MyBatis-Plus的save方法

#### 5.2.2 查询快递列表

```java
@Override
public Page<Express> getExpressList(Long stationId, String status, Integer pageNum, Integer pageSize) {
    Page<Express> page = new Page<>(pageNum, pageSize);
    LambdaQueryWrapper<Express> wrapper = new LambdaQueryWrapper<>();

    if (stationId != null) {
        wrapper.eq(Express::getStationId, stationId);
    }

    if (status != null && !status.isEmpty()) {
        wrapper.eq(Express::getStatus, Integer.parseInt(status));
    }

    wrapper.orderByDesc(Express::getCreateTime);
    return page(page, wrapper);
}
```

**业务逻辑说明：**
1. **分页设置**：创建分页对象
2. **条件构造**：使用LambdaQueryWrapper构建查询条件
3. **动态查询**：根据传入参数动态添加查询条件
4. **排序**：按创建时间降序排列
5. **分页查询**：返回分页结果

#### 5.2.3 取件码取件

```java
@Override
public Express pickupByCode(String pickupCode, Long userId) {
    if (pickupCode == null || pickupCode.trim().isEmpty()) {
        throw new RuntimeException("取件码不能为空");
    }

    LambdaQueryWrapper<Express> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Express::getPickupCode, pickupCode);
    Express express = getOne(wrapper);

    if (express == null) {
        throw new RuntimeException("取件码不存在");
    }

    if (express.getStatus() == 1) {
        throw new RuntimeException("该快递已取件");
    }

    express.setStatus(1);
    updateById(express);

    return express;
}
```

**业务逻辑说明：**
1. **参数校验**：检查取件码是否为空
2. **快递查询**：根据取件码查询快递信息
3. **状态检查**：验证快递是否已取件
4. **状态更新**：将状态更新为"已取件"
5. **返回结果**：返回更新后的快递信息

## 六、Controller层设计

### 6.1 管理员控制器

```java
@Tag(name = "管理员快递管理", description = "管理员快递录入、查询、更新、删除接口")
@RestController
@RequestMapping("/admin/express")
public class AdminExpressController {

    @Autowired
    private ExpressService expressService;

    @Operation(summary = "录入快递")
    @PostMapping("/add")
    public R<Express> addExpress(@Valid @RequestBody Express express) {
        try {
            Express result = expressService.addExpress(express);
            return R.success(result);
        } catch (RuntimeException e) {
            return R.error(e.getMessage());
        }
    }

    @Operation(summary = "快递列表查询")
    @GetMapping("/list")
    public R<Page<Express>> getExpressList(
            @Parameter(description = "快递站ID") @RequestParam(required = false) Long stationId,
            @Parameter(description = "快递状态") @RequestParam(required = false) String status,
            @Parameter(description = "页码") @RequestParam(defaultValue = "1") Integer pageNum,
            @Parameter(description = "每页大小") @RequestParam(defaultValue = "10") Integer pageSize) {
        try {
            Page<Express> page = expressService.getExpressList(stationId, status, pageNum, pageSize);
            return R.success(page);
        } catch (RuntimeException e) {
            return R.error(e.getMessage());
        }
    }
}
```

    @Autowired

## 七、工具类设计

### 7.1 取件码生成工具

```java
public class PickupCodeUtil {

    private static final Random random = new Random();

    public static String generatePickupCode() {
        int code = 100000 + random.nextInt(900000);
        return String.valueOf(code);
    }

    public static boolean validatePickupCode(String pickupCode) {
        if (pickupCode == null || pickupCode.length() != 6) {
            return false;
        }
        return pickupCode.matches("\\d{6}");
    }
}
```

### 7.2 功能说明

**generatePickupCode()方法：**
- 生成6位数字取件码
- 范围：100000-999999
- 确保取件码的唯一性和可读性

**validatePickupCode()方法：**
- 验证取件码格式
- 检查长度是否为6位
- 检查是否全部为数字

## 八、统一响应格式

### 8.1 R类设计

```java
@Data
public class R<T> {
    private int code;
    private String message;
    private T data;

    public static <T> R<T> success(T data) {
        R<T> r = new R<>();
        r.setCode(200);
        r.setMessage("成功");
        r.setData(data);
        return r;
    }

    public static <T> R<T> error(String message) {
        R<T> r = new R<>();
        r.setCode(500);
        r.setMessage(message);
        return r;
    }
}
```

## 十一、常见问题及解决方案

### 11.1 数据库ID自增问题

**问题描述：**
插入数据时报错："Field 'id' doesn't have a default value"

**解决方案：**
```sql
ALTER TABLE express MODIFY id BIGINT NOT NULL AUTO_INCREMENT;
```

### 11.2 外键约束问题

**问题描述：**
修改表结构时报错："Cannot change column 'id': used in a foreign key constraint"

**解决方案：**
```sql
ALTER TABLE pickup_record DROP FOREIGN KEY fk_pickup_express;
ALTER TABLE express DROP FOREIGN KEY fk_express_station;
```

### 11.3 取件码重复问题

**问题描述：**
生成的取件码可能与已存在的取件码重复

**解决方案：**
在生成取件码后，先查询数据库是否存在，如果存在则重新生成

```java
String pickupCode;
do {
    pickupCode = PickupCodeUtil.generatePickupCode();
} while (checkPickupCodeExists(pickupCode));
```
