## 二、项目包结构

```
com.school.campusexpress
 │
 ├─ config             // 配置类（Spring配置、Swagger、跨域、定时任务等）
 │
 ├─ controller         // 控制器层（接收请求，调用 Service 层）
 │
 ├─ service            // 业务逻辑层
 │   └─ impl           // Service 实现类
 │
 ├─ mapper             // 数据访问层（MyBatis Mapper接口）
 │
 ├─ entity             // 实体类，对应数据库表
 │
 ├─ dto                // 数据传输对象（请求对象）
 │
 ├─ vo                 // 视图对象（响应给前端的数据）
 │
 ├─ exception          // 异常处理类（全局异常、自定义异常）
 │
 ├─ interceptor        // 拦截器（权限校验、日志记录等）
 │
 ├─ annotation         // 自定义注解（权限注解、日志注解等）
 │
 ├─ util               // 工具类（日期、加密、文件操作等）
 │
 ├─ common             // 公共常量、枚举、基础类
 │
 └─ CampusExpressApplication.java   // 启动类
```

### 包结构说明

| 包名 | 用途 | 示例类 |
|------|------|--------|
| config | 存放所有配置类 | CorsConfig, MybatisPlusConfig, Knife4jConfig, WebMvcConfig |
| controller | 控制器层，处理HTTP请求 | UserController, ExpressController |
| service | 业务逻辑接口 | UserService, ExpressService |
| service.impl | 业务逻辑实现类 | UserServiceImpl, ExpressServiceImpl |
| mapper | MyBatis数据访问接口 | UserMapper, ExpressMapper |
| entity | 数据库实体类 | User, Express, PickupRecord, Station |
| dto | 数据传输对象（请求参数） | UserRegisterDTO, ExpressStorageDTO |
| vo | 视图对象（响应数据） | UserVO, ExpressVO |
| exception | 异常处理类 | GlobalExceptionHandler, 自定义异常类 |
| interceptor | 拦截器 | JwtInterceptor, LogInterceptor |
| annotation | 自定义注解 | RequireAuth, LogAnnotation |
| util | 工具类 | JwtUtil, PasswordUtil, DateUtil |
| common | 公共类 | R（统一响应类）、枚举、常量 |

## 三、数据库表结构

### 1. user表 - 用户表
```sql
CREATE TABLE user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
  password VARCHAR(255) NOT NULL COMMENT '密码(加密)',
  real_name VARCHAR(50) COMMENT '真实姓名',
  phone VARCHAR(20) COMMENT '手机号',
  role VARCHAR(20) DEFAULT 'user' COMMENT '角色(admin/user)',
  status INT DEFAULT 1 COMMENT '状态(0=禁用,1=启用)',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='用户表';
```

### 2. express表 - 快递表
```sql
CREATE TABLE express (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '快递ID',
  express_no VARCHAR(50) NOT NULL UNIQUE COMMENT '快递单号',
  recipient_name VARCHAR(50) NOT NULL COMMENT '收件人姓名',
  recipient_phone VARCHAR(20) NOT NULL COMMENT '收件人手机号',
  pickup_code VARCHAR(10) NOT NULL COMMENT '取件码',
  status VARCHAR(20) DEFAULT 'pending' COMMENT '状态(pending=待取件,picked=已取件,expired=超期)',
  storage_time DATETIME NOT NULL COMMENT '入库时间',
  pickup_time DATETIME COMMENT '取件时间',
  station_id BIGINT COMMENT '快递站ID',
  admin_id BIGINT COMMENT '操作管理员ID',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='快递表';
```

### 3. pickup_record表 - 取件记录表
```sql
CREATE TABLE pickup_record (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '记录ID',
  express_id BIGINT NOT NULL COMMENT '快递ID',
  user_id BIGINT NOT NULL COMMENT '取件用户ID',
  pickup_time DATETIME NOT NULL COMMENT '取件时间',
  pickup_code VARCHAR(10) NOT NULL COMMENT '取件码',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) COMMENT='取件记录表';
```

### 4. station表 - 快递站表
```sql
CREATE TABLE station (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '快递站ID',
  station_name VARCHAR(100) NOT NULL COMMENT '快递站名称',
  location VARCHAR(200) COMMENT '位置',
  admin_id BIGINT COMMENT '负责人ID',
  status INT DEFAULT 1 COMMENT '状态(0=禁用,1=启用)',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) COMMENT='快递站表';
```
