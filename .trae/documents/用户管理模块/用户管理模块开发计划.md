## 📝 注意事项

1. **权限验证**：所有管理员接口必须验证JWT中的role字段为admin
2. **密码安全**：重置密码时使用默认密码123456，并进行加密存储
3. **分页查询**：使用MyBatis-Plus的Page对象实现分页
4. **数据完整性**：删除用户前需要检查是否有关联数据（如快递记录）
5. **状态管理**：禁用用户后，该用户无法登录系统
6. **角色管理**：管理员可以添加普通用户和管理员，但不能添加超级管理员
7. **异常处理**：提供友好的错误提示信息
8. **代码规范**：遵循项目核心规则，代码简洁易懂

### 🎯 答辩加分设计要点

#### 1. 管理员不能删除自己
**答辩场景**：老师很可能随口问一句："管理员能不能把自己删掉？"

**实现方式**（Service层）：
```java
if (id.equals(currentUserId)) {
    throw new RuntimeException("不能删除当前登录用户");
}
```

**答辩话术**："系统设计了安全机制，管理员不能删除自己，防止误操作导致系统无法管理。"

---

#### 2. 重置密码只允许重置普通用户
**实现方式**：
```java
if ("admin".equals(user.getRole())) {
    throw new RuntimeException("不能重置管理员密码");
}
```

**答辩话术**："为了系统安全，管理员只能重置普通用户的密码，不能重置其他管理员的密码，防止权限滥用。"

---

#### 3. 搜索接口内部复用list逻辑
**设计思路**：
- 搜索功能是在分页查询基础上的扩展
- search = list + keyword 条件

**实现方式**：
```java
public Page<User> searchUsers(Integer pageNum, Integer pageSize, String keyword) {
    Page<User> page = new Page<>(pageNum, pageSize);
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.like(StringUtils.isNotBlank(keyword), User::getUsername, keyword)
           .or()
           .like(StringUtils.isNotBlank(keyword), User::getPhone, keyword);
    return page(page, wrapper);
}
```

**答辩话术**："搜索功能是在分页查询基础上的扩展，通过动态条件查询实现，代码复用率高，维护性好。"

### 6. 重置密码时检查角色
```java
public boolean resetPassword(Long id) {
    User user = getById(id);
    if (user == null) {
        throw new RuntimeException("用户不存在");
    }
    if ("admin".equals(user.getRole())) {
        throw new RuntimeException("不能重置管理员密码");
    }
    String defaultPassword = "123456";
    user.setPassword(PasswordUtil.encode(defaultPassword));
    return updateById(user);
}
```
