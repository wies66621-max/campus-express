## 📝 注意事项

### ⚠️ 必须注意的5个真实坑

#### 1️⃣ Express 实体类字段映射一定要写清楚
```java
@TableName("express")
public class Express {

    @TableId(type = IdType.AUTO)
    private Long id;

    @TableField("tracking_number")
    private String trackingNumber;

    @TableField("pickup_code")
    private String pickupCode;

    @TableField("station_id")
    private Long stationId;
}
```
❗ 不写 @TableField，迟早再翻一次车

---

#### 2️⃣ application.yml 确认开启驼峰映射
```yaml
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
```
否则 pickup_code → pickupCode 可能不稳定

---

#### 3️⃣ 批量操作方法必须加事务
```java
@Transactional
public boolean batchUpdateStatus(List<Long> ids, Integer status) {
    // 批量更新逻辑
}
```
答辩理由：批量操作属于原子业务，需要事务保证一致性

---

#### 4️⃣ pickup_code 字段：建议加普通索引（不是唯一）
```sql
ALTER TABLE express ADD INDEX idx_pickup_code (pickup_code);
```
❗ 不要唯一索引（6位数字不安全）

---

#### 5️⃣ Controller 返回统一用 R（别直接 return boolean）
```java
@PostMapping("/batch-delete")
public R<String> batchDelete(@RequestBody List<Long> ids) {
    boolean success = expressService.batchDelete(ids);
    if (success) {
        return R.success("批量删除成功");
    }
    return R.error("批量删除失败");
}
```
这是老师最常扣分的小细节之一

### 🎯 答辩加分设计要点

#### 1. 取件码唯一性保证机制
**答辩场景**：老师可能会问："取件码会不会重复？"

**实现方式**（Service层）：
```java
String pickupCode;
int maxRetries = 10;
int retryCount = 0;
LambdaQueryWrapper<Express> codeWrapper;
do {
    pickupCode = PickupCodeUtil.generate();
    codeWrapper = new LambdaQueryWrapper<>();
    codeWrapper.eq(Express::getPickupCode, pickupCode);
    retryCount++;
} while (getOne(codeWrapper) != null && retryCount < maxRetries);

if (retryCount >= maxRetries) {
    throw new RuntimeException("生成取件码失败，请重试");
}
```

**答辩话术**："系统设计了取件码唯一性保证机制，生成取件码时会检查数据库中是否已存在，如果重复会重新生成，最多重试10次，确保取件码的唯一性。"

---

#### 2. 防止重复取件机制
**实现方式**：
```java
if (express.getStatus() != 0) {
    throw new RuntimeException("快递已被取件");
}
```

**答辩话术**："系统设计了防止重复取件的机制，取件时会检查快递状态，只有待取件状态的快递才能取件，已取件的快递无法再次取件。"

---

#### 3. 取件记录只在取件成功后生成
**实现方式**：
```java
@Transactional
public Express pickupByCode(String pickupCode, Long userId) {
    LambdaQueryWrapper<Express> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Express::getPickupCode, pickupCode);
    Express express = getOne(wrapper);

    if (express == null) {
        throw new RuntimeException("取件码不存在");
    }

    if (express.getStatus() != 0) {
        throw new RuntimeException("快递已被取件");
    }

    express.setStatus(1);
    express.setUpdateTime(LocalDateTime.now());
    updateById(express);

    PickupRecord record = new PickupRecord();
    record.setExpressId(express.getId());
    record.setUserId(userId);
    record.setPickupCode(pickupCode);
    record.setPickupTime(LocalDateTime.now());
    record.setCreateTime(LocalDateTime.now());
    pickupRecordService.save(record);

    return express;
}
```

**答辩话术（必杀）**："取件记录不是冗余数据，而是行为日志，只在取件成功后生成。这样可以追踪每一次取件操作，方便后续的数据分析和问题排查。"

#### 4. 快递统计功能（只统计express表，不JOIN）
**实现方式**：
```java
public Map<String, Object> getStatistics() {
    Map<String, Object> statistics = new HashMap<>();

    LambdaQueryWrapper<Express> pendingWrapper = new LambdaQueryWrapper<>();
    pendingWrapper.eq(Express::getStatus, 0);
    long pendingCount = count(pendingWrapper);

    LambdaQueryWrapper<Express> pickedWrapper = new LambdaQueryWrapper<>();
    pickedWrapper.eq(Express::getStatus, 1);
    long pickedCount = count(pickedWrapper);

    statistics.put("pendingCount", pendingCount);
    statistics.put("pickedCount", pickedCount);
    statistics.put("totalCount", pendingCount + pickedCount);

    return statistics;
}
```

**答辩话术**："系统提供了快递统计功能，可以实时统计待取件数量、已取件数量和总数量，帮助管理员了解快递处理情况，提升管理效率。统计功能只查询express表，不进行复杂的联表操作，保证查询性能。"

---

#### 5. 批量操作功能（加事务）
**实现方式**：
```java
@Transactional
public boolean batchDelete(List<Long> ids) {
    if (ids == null || ids.isEmpty()) {
        throw new RuntimeException("删除ID列表不能为空");
    }
    return removeByIds(ids);
}

@Transactional
public boolean batchUpdateStatus(List<Long> ids, Integer status) {
    if (ids == null || ids.isEmpty()) {
        throw new RuntimeException("ID列表不能为空");
    }
    if (status == null || (status != 0 && status != 1)) {
        throw new RuntimeException("状态值不正确");
    }

    LambdaUpdateWrapper<Express> wrapper = new LambdaUpdateWrapper<>();
    wrapper.in(Express::getId, ids)
           .set(Express::getStatus, status)
           .set(Express::getUpdateTime, LocalDateTime.now());

    return update(wrapper);
}
```

**答辩话术**："系统提供了批量操作功能，支持批量删除和批量更新快递状态，提升了管理员的操作效率，减少了重复操作。批量操作使用了事务注解，保证操作的原子性和一致性。"
