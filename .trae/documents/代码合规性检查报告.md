## 四、发现的问题

### ⚠️ 问题1：密码比较未使用加密（中风险）

**位置**: [UserServiceImpl.java](file:///d:\IDEAProjects\campus-express\src\main\java\com\school\campusexpress\service\impl\UserServiceImpl.java#L99)

**问题代码**:
```java
if (!password.equals(user.getPassword())) {
    throw new RuntimeException("密码错误");
}
```

**问题描述**:
- 直接比较明文密码，未使用PasswordUtil.matches()方法
- 虽然有PasswordUtil工具类，但未在登录时使用

**影响**:
- 密码安全性降低
- 答辩时老师可能会质疑为什么有加密工具但不使用

**建议修改**:
```java
if (!PasswordUtil.matches(password, user.getPassword())) {
    throw new RuntimeException("密码错误");
}
```

**优先级**: 中
**预计修改时间**: 5分钟

---

### ⚠️ 问题2：注册时密码未加密（中风险）

**位置**: [UserServiceImpl.java](file:///d:\IDEAProjects\campus-express\src\main\java\com\school\campusexpress\service\impl\UserServiceImpl.java#L77)

**问题代码**:
```java
user.setPassword(password);
```

**问题描述**:
- 注册时直接存储明文密码
- 未使用PasswordUtil.encode()方法加密

**影响**:
- 密码安全性降低
- 不符合安全最佳实践

**建议修改**:
```java
user.setPassword(PasswordUtil.encode(password));
```

**优先级**: 中
**预计修改时间**: 5分钟

---

### ⚠️ 问题3：JWT密钥管理可以简化（低风险）

**位置**: [JwtUtil.java](file:///d:\IDEAProjects\campus-express\src\main\java\com\school\campusexpress\util\JwtUtil.java#L17)

**问题代码**:
```java
private static final Key key = Keys.hmacShaKeyFor(SECRET_KEY.getBytes());
```

**问题描述**:
- 使用了Keys.hmacShaKeyFor()生成密钥
- 虽然更安全，但增加了理解难度

**影响**:
- 答辩时需要解释Key的概念
- 对于毕设项目可能过于复杂

**建议修改（可选）**:
```java
private static final String SECRET_KEY = "campus-express-secret-key-for-jwt-token-generation-2024";
```

**优先级**: 低
**是否必须修改**: 否，当前实现也可以接受

## 六、技术选型三问检查

### 1. MyBatis-Plus

**为什么要用它？**
- 简化数据库操作，减少重复代码
- 提供分页、条件构造等常用功能

**不用它会有什么问题？**
- 需要编写大量重复的SQL语句
- 分页、条件查询等需要自己实现
- 开发效率低，代码冗余

**它在本系统中具体解决了什么？**
- 快速实现用户、快递等模块的CRUD操作
- 实现用户列表的分页查询
- 实现按用户名、手机号等条件的筛选查询

**结论**: ✅ 符合要求

---

### 2. JWT

**为什么要用它？**
- 无状态的身份认证机制
- 适合前后端分离架构

**不用它会有什么问题？**
- 需要使用Session，不适合前后端分离
- 需要服务器存储会话信息
- 扩展性差

**它在本系统中具体解决了什么？**
- 实现用户登录后的身份认证
- 保护需要登录才能访问的接口
- 前端通过token识别用户身份

**结论**: ✅ 符合要求

---

### 3. BCrypt

**为什么要用它？**
- 密码加密存储，提高安全性
- 防止密码泄露

**不用它会有什么问题？**
- 密码明文存储，安全性低
- 数据库泄露会导致密码泄露
- 不符合安全最佳实践

**它在本系统中具体解决了什么？**
- 用户密码加密存储
- 防止密码被直接查看
- 提高系统安全性

**结论**: ✅ 符合要求

---

### 4. Knife4j

**为什么要用它？**
- 自动生成API文档
- 提供在线测试接口功能

**不用它会有什么问题？**
- 需要手动编写API文档
- 文档更新不及时
- 接口测试不方便

**它在本系统中具体解决了什么？**
- 自动生成用户管理、快递管理等模块的接口文档
- 方便老师查看和测试接口
- 便于答辩时演示接口功能

**结论**: ✅ 符合要求
